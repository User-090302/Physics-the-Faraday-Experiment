# Учебный проект

## Назначение

Проект создан в **образовательных целях**.  
В репозитории уже есть файлы с готовыми и корректными данными в формате JSON.  
Проект рассчитан на пользователей как с техническим бэкграундом, так и без него.  
Основная цель — демонстрация работы с данными без необходимости изучения внутренней логики кода.

---

## Данные

Все JSON-файлы являются полностью готовыми и проверенными.  
Они представляют собой исходные данные, с которыми работает проект.  
Редактирование возможно, так как исходники распространяются под **GPL-3.0**.


---
# Описание:

Как известно из лабораторной, она состоит из 2х частей:

1. Калибровка(фвйл `kalibr(no gen).pyw`ли `kalibr.no.gen.exe`) - записать показания с БП элеткро магнита(I в Амперах) и показатель магнитной индукции(B в мТл), при разных расстояниях наконечников(в данном случае 20мм и 40мм, рекомендуется менять местами в таблицах, т.к. электромагнит ведет себя крайне неадекватно), и построить соответствующий график. Данная программа всё это делает и сохраняет данные в `**.json`, и автоматически сохраняет график в `.png` формат

2. Даьнейший эксперимент с построением Верде(`exp.ver-0.1.release.pyw` или (`exp.ver-0.1.release.exe`) - записать измение света с точками максимума поляризированного света, проходящего через диэлектрик, записываются в основном данные I(А),А1 и А2. B вычисляется относительно каллибровки. Данный файл принимает данные из калибровочного json файла проводя через математичкую модель(за счёт этого можно значение I в калибровке и дальнейшем эксперименте **не обязательно должны совпадать**) и производятся необходимые вычисления и построение графика. Опять же, всё сохраняется в том же json и png виде, как и при каллибровке.

Т.к. такой вариант неудобный в виду требования для оформления(с обязательным заполнением расчетов) товарища Дмитрия Сергеевича, существует файл `raschets.py`(он был не скомпилирован т.к. было лень писать графический интерфейс для вспомогательного скрипта записи расчетов а так. же за счет разнящихся названий файлов, ввиду чего приходилось бы писать код для ввода пути к файлу, если это надо - редактируйте и компилируйте сами, не зря же стоит GPL лицензия), что принимает финальные вычисления и даписывает их в удобном(насколько можно так сказать) виде, достаточным, чтобы можно было переписать в заготовку.

## Требования

Для работы проекта необходимо установить Python 3.11+ и следующие библиотеки:  

- `numpy`  
- `pandas`  
- `scikit-learn`  
- `matplotlib`
